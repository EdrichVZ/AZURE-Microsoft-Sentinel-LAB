# AZURE-Microsoft-Sentinel-LAB
Implemented Microsoft Sentinel, Microsoft's cloud-native Security Information and Event Management (SIEM) and Security Orchestration, Automation, and Response (SOAR) solution. I connected data sources, configured analytics rules, created custom alerts, deployed a playbook for automated incident response, and investigated security incidents.

##Overview:
Security teams require centralized visibility and automated response capabilities to effectively detect and respond to threats. In this lab, I onboarded Microsoft Sentinel to a Log Analytics Workspace, connected Azure Activity logs, configured analytics rules, and created a Logic App playbook to automate incident management. I then generated a security event and observed how Sentinel detected the activity, created an incident, and triggered an automated response workflow. This provided hands-on experience with the core components of a Security Operations Center (SOC) solution in Azure.

##Objectives(Completed):
- Task 1: On-board Microsoft Sentinel
- Task 2: Connect Azure Activity to Sentinel
- Task 3: Create a rule that uses the Azure Activity data connector.
- Task 4: Create a playbook
- Task 5: Create a custom alert and configure the playbook as an automated response.

##Lessons Learned:
- Onboard Microsoft Sentinel and connect it to a Log Analytics Workspace.
- Configure data connectors to ingest Azure Activity logs.
- Create and manage analytics rules for threat detection.
- Generate security alerts based on specific activities and events.
- Create and deploy playbooks using Azure Logic Apps.
- Automate incident response using SOAR capabilities.
- Investigate and manage incidents within Microsoft Sentinel.
- Understand the relationship between data collection, analytics, incidents, and automated responses.
- Use Microsoft Sentinel as a centralized platform for security monitoring and threat management.
